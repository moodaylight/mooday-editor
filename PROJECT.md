# MOODAY 开发避坑总纲

# 最大技术坑（必须避免）

# 1. script.js 爆炸

问题：
功能越来越多后会失控。

解决：
后期模块化。

但：
当前阶段不要急拆。

---

# 2. 写死参数

问题：
横版和多尺寸会崩。

解决：
所有尺寸统一 config 管理。

不要到处写：

frameWidth = 12

---

# 3. 横版复制代码

问题：
维护崩溃。

解决：
配置化支持横版。

不要：
复制一套横版代码。

---

# 4. Retina 模糊

问题：
iPhone 高清屏会糊。

解决：
必须做 DPR 适配。

后续必须升级 canvas。

---

# 5. 用户大图崩溃

问题：
iPhone 图片太大导致卡死。

解决：

编辑器：
使用压缩预览图。

最终导出：
使用原图。

---

# 6. 预览图不等于生产图

问题：
实际打印会偏暗。

解决：

Preview Render：
给用户看。

Production Render：
给打印机。

后续必须双渲染。

---

# 7. AI 修改冲突

问题：
局部修改容易崩。

解决：

小改：
局部替换。

大改：
整文件覆盖。

---

# 8. 没有稳定版本

问题：
一次改崩全没。

解决：

每完成一个稳定功能：

GitHub Commit

例如：

v0.5 UI优化
v0.6 灯光系统

---

# 9. UI越来越乱

问题：
按钮越来越多。

解决：

功能分层。

一级：

文字
灯光
图层

二级：

颜色
发光
删除

---

# 10. 太早做复杂功能

暂时不要：

AI美化
自动抠图
复杂模板
云工程
复杂后端

当前只做：

稳定
好用
高级感
