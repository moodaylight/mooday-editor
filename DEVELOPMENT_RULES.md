# MOODAY 开发规范（第一版）

一、架构优先，功能第二。

二、凡是未来一定需要拆分的模块，必须提前拆分，不允许等代码庞大以后再重构。

三、一个功能，全项目只能存在一个英文名称。

四、一个文件，只允许承担一个明确职责。

五、所有模块统一采用：

Config

↓

Editor

↓

Render

三层结构。

六、Render 层只允许负责绘制。

禁止修改 State。

禁止处理按钮逻辑。

禁止处理业务逻辑。

七、State 统一管理。

所有 currentXXX 状态变量统一放入 state.js。

八、所有开发文档统一采用中文。

九、所有备份采用追加方式，不覆盖历史内容。

十、每次备份必须记录日期。

十一、技术负责人必须主动进行架构巡检。

发现未来风险时，应主动提出，不等待问题发生后再处理。

十二、所有开发必须以项目长期维护和长期扩展为目标，而不是仅满足当前需求。
